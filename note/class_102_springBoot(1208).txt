애플리케이션 테스트 -
본래는 소프트웨어 구현 외에 테스트 단계가 존재. 개발자와 테스터도 별개로 존재
테스트는 총 단위(로직) - 통합(사용자 시스템과 비슷하게) - 시스템 - 인수 테스트(인수자가 인수받은 후 테스트)

테스트 방법 - 1) 구조기반 화이트박스 테스트 : 프로그램 내부 구조 검증 (제어흐름,조건결정)
2) 명세기반 블랙박스 테스트 : 실행기반 (기능 및 실행되는지만 보는것)

테스트 자동화도구 - 시간/인력투입을 최소화, 단 모든 테스트과정 자동화는 불가

테스트 수행순서
1) 테스트 계획서 작성 (케이스 예시 확인, 목적과 범위 확인, 데이터 확인)
2) 기능 단위 테스트


결함 - 기록 후 검토 (개발자) 수정 - 재확인(피드백-수정) - 최종 분석 및 보고서
결함관리 - 기능명세서(유스케이스 다이어그램 등) 명시된 동작이 수행되지 않거나,
불가능하다고 명시된 동작이 수행될 때. 명시되어 있지 않은 동작이 수행되는 경우
명시되진 않았지만 수행되야 할 동작을 수행하지 않는 경우 / 기타 테스터 시각에 문제가 있다 판단하는 경우
(ex이해어렵 / 사용어렵 / 느림)
