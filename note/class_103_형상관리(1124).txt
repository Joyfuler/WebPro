실제 배포될 소프트웨어(사용자에게 공급될)는 버전관리(형상관리) 필요 ex) 깃 버전관리
소프트웨어의 변경 내용을 관리하는 용도

장점
1) 소스코드를 저장하거나 공유 가능
2) 버전관리 - 이전 버전 / 새 버전 정보 접근 가능
3) 협업공간 존재

평소 사용하는 명령어 외에 git reset / git revert등으로 이전버전으로 돌아갈 수 있음.

Git - 분산저장소 방식
SVN(Subversion) - 프로젝트 수정 사항이 즉시 적용됨.(서버에 있는 파일이 변조될 위험성 존재)
이후 동기화하면 하드에 있는 파일 역시 변조 / 휘발될 수 있으므로 주기적인 백업이 필요
