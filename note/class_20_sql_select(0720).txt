sql에서 사용하는 연산자 정리
DISTINCT - 필드 중 중복되는 데이터를 제거.
BETWEEN A AND B : A와 B 사이의 값을 출력. NOT BETWEEN을 적을 경우, A보다 낮고 B보다 큰 값을 출력 가능.
IN : or를 편하게 사용할 수 있는 방법. 
LIKE: 검색으로 가장 많이 사용하는 방법. _ 는 한 글자, %는 0글자 이상. 
A가 들어가는 필드를 찾고 싶다면 WHERE (필드명) LIKE '%A%' 식으로 검색. 
IS NULL : NULL값이 있는 필드를 찾음. 자바의 경우 NULL은 0으로 리턴되므로 필요.
정렬 명령어인 ORDER BY 는 기본 오름차순(ASC). 내림차순으로 정렬하고 싶다면 ORDER BY (필드명) DESC로 작성.
--JOIN
EQUI JOIN - 다른 테이블에 같은 값이 있는 경우, 값이 겹치는 곳을 =으로 합쳐져 필요 없는 데이터가 나오지 않도록 조절.
EMP.DEPTNO = DEPT.DEPTNO 형식으로 합치면 필요없는 데이터를 필터링 가능.
NON-EQUI JOIN - 조인된 조건이 특정 범위에 있는지 조사하기 위해 사용. ex) 받는 급여가 몇 등급에 속하는지를 함께 출력
겹치는 필드가 없으므로 =는 필요하지 않지만, 필요 없는 데이터가 나오지 않도록 BETWEEN A AND B 등 WHERE(쿼리) 설정이 중요.
